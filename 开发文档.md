# MotorLearn 电机知识学习平台 · 开发文档

> 面向自动化学生 / 嵌入式单片机开发者的电机控制知识学习网站。纯前端静态站点，无需后端、无需构建，双击 `index.html` 即可运行，也可部署到 GitHub Pages。
>
> 本文档面向后续维护者（含 AI 协作者），说明：项目定位、技术选型、整体架构、内容数据层如何扩展、图表系统、代码规范，以及后续可补充方向。

---

## 一、项目定位

| 项目        | 说明 |
|-----------|------|
| 名称        | MotorLearn（电机知识学习平台） |
| 目标用户    | 自动化 / 机电 / 电子专业学生、嵌入式单片机开发者（STM32/Arduino/ESP32 等） |
| 核心价值    | 把分散在课本、博客、数据手册里的电机控制知识，**系统化、可视化、可交互、可检索**地组织在一个网页里 |
| 运行方式    | 纯静态：浏览器打开 `index.html`，或部署到 GitHub Pages 等静态托管 |
| 是否需要后端| 否。所有数据持久化用浏览器 `localStorage` |
| 线上地址    | GitHub Pages（main 分支自动部署） |

网站采用**单页应用（SPA）**模式：一个 `index.html` 外壳 + JS 动态渲染内容区。导航点击不刷新页面，靠 JS 切换内容。

**内容覆盖全景**：入门篇（电磁学/参数/驱动）→ 进阶篇（PID/编码器/BLDC换向/FOC/坐标变换/无感/多环/伺服通信）→ 电机分类（有刷/无刷/步进/伺服/舵机，含 STM32 代码）→ 机器人应用（运动学/轨迹规划/MCU-ROS 桥接）→ 工具箱（对比/计算器/协议校验/进度）。

## 二、技术栈与选型思路

### 选型原则
1. **零依赖、零构建**：不引入 React/Vue/Webpack，保证任何人 clone 下来就能改、就能跑。
2. **CDN 即用**：只引用必须的外部库（Tailwind / ECharts / KaTeX），全部走 CDN。
3. **数据驱动**：所有内容写在 `js/data.js` 一个大对象里，UI 只负责把数据渲染成 HTML，互不耦合。

### 具体技术清单

| 类别 | 技术 | 用途 | 选型理由 |
|------|------|------|---------|
| 样式 | Tailwind CSS（CDN） | 原子化布局类 | 写得快、不用维护 class 名 |
| 主题 | 原生 CSS 变量 | 暖色纸张主题 + 深色模式 | 切换 `data-theme`，变量自动替换 |
| 公式 | KaTeX + auto-render | 渲染 `$$F=BIL$$` 物理/数学公式 | 电机控制绕不开公式，比 MathJax 快 |
| 图表 | ECharts 5 | 波形图、知识图谱、进度饼图 | 中文文档好、交互性强、可 hover 看数值 |
| 交互组件 | 原生 JS + SVG | H桥交互图、机械臂拖拽 | 不引框架，DOM 事件 + SVG 精确控制 |
| 路由 | 原生 JS | SPA 路由、手风琴、Tab | 不引框架，体量小 |
| 存储 | localStorage | 收藏、进度、答题、主题 | 无需后端的"用户态" |

### 脚本加载顺序（关键！）
`index.html` 底部按依赖顺序加载，**顺序不能乱**：

```
data.js       ← 内容数据（定义全局 MotorData、QuizData、AllKnowledgeIds）
  ↓
progress.js   ← 进度模块（依赖 MotorData）
favorites.js  ← 收藏 + 浏览历史（依赖 MotorData）
search.js     ← 搜索模块（依赖 MotorData）
calculator.js ← 公式计算器
charts.js     ← ECharts 图表 + 交互组件（依赖 Progress、navigateTo）
validator.js  ← 协议校验工具
quiz.js       ← 自测练习（依赖 QuizData、navigateTo）
  ↓
app.js        ← 主控制器，最后加载，使用以上所有模块
```

> ⚠️ 历史教训：曾因 `quiz.js` 漏挂 `<script>`，导致所有带测验的知识详情页抛 `ReferenceError` 白屏。新增 JS 模块务必同步更新 `index.html` 引用，且在 `app.js` **之前**。

## 三、整体架构

### 目录结构
```
电机学习/
├── index.html              # SPA 外壳：导航栏、侧边栏、内容容器、script 引用
├── css/
│   └── style.css           # CSS 变量、组件样式（code-block/svg-figure/手风琴/测验/校验）
├── js/
│   ├── data.js             # ⭐ 所有内容数据（约 3000 行，项目的"数据库"）
│   ├── app.js              # ⭐ 主控制器（路由、页面渲染、交互绑定、图表调度）
│   ├── progress.js         # 学习进度（pending/learning/completed）
│   ├── favorites.js        # 收藏 + 浏览历史
│   ├── search.js           # 全文搜索 + 结果跳转（含手风琴展开）+ 关键词高亮
│   ├── quiz.js             # 自测练习渲染 + 答题记录
│   ├── calculator.js       # 公式计算器（KV、步距角、功率、PWM占空比等）
│   ├── charts.js           # ⭐ ECharts 波形图 + 交互组件（H桥/机械臂）+ 知识图谱/进度
│   └── validator.js        # CRC8/16/32、异或、取反校验工具
├── assets/
│   └── icons/头像.jpg       # 顶栏 logo
├── .gitignore              # 忽略日志、.claude/ 等
└── 开发文档.md             # 本文档
```

### 运行时数据流

```
┌─────────────┐    渲染     ┌──────────────┐
│  data.js    │ ─────────▶ │   app.js     │ ──▶ innerHTML 注入 #page-container
│ (MotorData) │            │ (renderPage) │
└─────────────┘            └──────┬───────┘
      ▲                            │ 渲染后调度
      │                            ▼
      │                   ┌──────────────────┐
      │                   │ Charts.renderAll │ ← 扫描 [data-chart] 自动渲染
      │                   └────────┬─────────┘
      │                            │
      └──── 读引用 ───────┌────────┴────────┐
                          ▼                 ▼
                   ┌────────────┐    ┌──────────────┐
                   │ ECharts波形 │    │ 交互SVG组件  │
                   │(PWM/霍尔等) │    │(H桥/机械臂)  │
                   └────────────┘    └──────────────┘
                          │
                          ▼ 读写
                   ┌──────────────────┐
                   │ localStorage     │
                   │ (进度/收藏/答题) │
                   └──────────────────┘
```

**核心渲染函数** `renderPage(pageId)`（位于 `app.js`）根据 `pageId` 分发：

| pageId | 渲染函数 | 说明 |
|--------|---------|------|
| `home` | `renderHomePage()` | 首页统计 + 快速入口 + 速览表 |
| `beginner` / `advanced` / `robotics` | `renderSectionPage()` | 章节卡片列表（三个分组共用） |
| `motors` | `renderMotorsPage()` | 5 种电机卡片列表 |
| `industry` | `renderIndustryPage()` | 行业科普手风琴 |
| `roadmap` | `renderRoadmapPage()` | 知识图谱 + 书单 + 选型 |
| `tools` | `renderToolsPage()` | 对比 / 计算器 / 校验 / 进度 |
| `beginner-em` 等知识点 id | `renderDetailPage()` | 知识点详情（含测验） |
| `bldc` 等电机 id | `renderMotorPage()` | 电机详情（手风琴 + 测验） |

> `findSection(id)` 会依次在 beginner / advanced / robotics 三个分组的 sections 里查找，支持跨分组。新增分组需在此函数加一行。

## 四、数据层结构（最常改的部分）

所有内容都在 `js/data.js` 的 `MotorData` 对象里。**添加新内容 = 往对应字段追加对象**。

### 4.1 导航 `nav`
侧边栏来源。支持一级菜单和可折叠二级菜单。当前有 6 个分组：首页、入门篇、进阶篇、电机分类、机器人应用、电机行业、学习路径、工具箱。

```js
nav: [
  { id: 'home', label: '首页', icon: 'home' },
  {
    id: 'advanced', label: '进阶篇', icon: 'rocket',
    badge: '进阶', badgeClass: 'badge-advanced',
    children: [ { id: 'advanced-pid', label: 'PID控制理论' }, /* ... */ ]
  },
  {
    id: 'robotics', label: '机器人应用', icon: 'rocket',  // 新增分组示例
    children: [ { id: 'kinematics', label: '运动学入门' }, /* ... */ ]
  },
]
```

> **添加导航项后，必须在对应数据结构准备内容**，否则点击回退首页。新增**分组**还需在 `app.js` 的 `renderPage` 加 case + `findSection` 加查找分支。

### 4.2 知识章节 `beginner/advanced/robotics.sections`
三个分组结构一致，每个 section 是知识点详情页：

```js
{
  id: 'advanced-pid-impl',        // 必须与 nav.children.id 一致
  title: 'PID的C语言实现',
  desc: '位置式/增量式/抗饱和...',  // 列表卡片简介
  icon: '💻',
  tags: ['核心算法', '工程实现'],
  content: `                       // 详情页正文，支持 HTML
    <h3>...</h3>
    <div class="code-block">...</div>
    <div class="formula-block">$$...$$</div>
    <div data-chart="pwm" class="chart-container" style="height:300px"></div>
  `
}
```

### 4.3 电机类型 `motorTypes`
每种电机是一个 key（与 nav.children.id 对应），含 specs 参数表 + sections 手风琴：

```js
motorTypes: {
  'bldc': {
    title: '无刷直流电机 (BLDC)',
    subtitle: '...', icon: '🌀', overview: '...',
    specs: { 电压: '...', 转速: '...', 效率: '...' },
    sections: [ { title: '工作原理', content: `...` }, /* ... */ ]
  },
}
```

### 4.4 自测题 `QuizData`
按 section id 索引。电机类用 `'motor-' + 电机id`：

```js
const QuizData = {
  'advanced-pid-impl': [
    { question: '...', options: ['A','B','C','D'], answer: 1, explanation: '...' },
  ],
  'motor-stepper': [ /* ... */ ],   // 电机页的测验
};
```

> 5 种电机现在全部有测验。`Quiz.answer()` 里会自动剥离 `motor-` 前缀再导航，避免答题后跳错页。

### 4.5 其他数据
- `home.intro`：网站介绍文字（显示在首页 hero 下方）
- `home.features`：4 大特色卡片（系统知识/可运行代码/交互图表/在线沙盒）
- `home.stats`：首页统计数字（**必须真实**，曾因"实战项目6"不实被修正为 17/5/30+/6）
- `home.quickStart`：6 个快速入口卡片
- `industry` / `compareTable` / `roadmap`（phases/books/devboards）/ `icons`

## 五、图表与交互系统（重要）

### 5.1 ECharts 波形图（4 种）+ Python 仿真沙盒（1 种）
在 `charts.js` 的 `_charts` 对象里，content 用占位 div 触发：

```html
<div data-chart="pwm" class="chart-container" style="height:300px"></div>
<div data-chart="python-sim" class="chart-container" style="min-height:480px"></div>
```

可用类型：`pwm`（PWM占空比）、`hall`（霍尔三路波形）、`encoder`（AB正交）、`trajectory`（梯形/S曲线）、`python-sim`（Pyodide浏览器Python仿真沙盒，含代码编辑器和matplotlib画图）。

**调度机制**：`app.js` 渲染完页面后调用 `Charts.renderAll('page-container')`，扫描所有 `[data-chart]` 元素自动渲染（带 `data-init` 防重复）。**新增图表类型只需在 `_charts` 加函数 + content 放占位 div，无需改 app.js。**

### 5.2 交互式 SVG 组件（2 种）
- `data-chart="hbridge"`：H桥电路图，4 个状态按钮（正转/反转/制动/滑行），点击切换 MOS 管高亮 + 电流流动动画 + 电机旋转方向动画
- `data-chart="arm"`：2-DOF 机械臂，拖拽末端（或点击空白）实时显示 θ1/θ2/x/y，体现正逆运动学，角度弧跟随标注

这两个是纯 SVG + DOM 事件实现（非 ECharts），也在 `_charts` 对象里，通过同一个 `renderAll` 调度。

### 5.3 知识图谱与进度图
- `Charts.renderKnowledgeGraph(id)`：roadmap 页的关系图（ECharts graph）
- `Charts.renderProgressChart(id)`：tools 页的进度饼图

## 六、代码规范（重要约定）

### 6.1 代码示例的两类写法

| 类型 | 场景 | 写法 | 示例 |
|------|------|------|------|
| **纯算法** | 讲解算法原理（PID/Clarke/Park/SVPWM/IK/CRC/SMO） | 纯 C 函数，平台无关，外设操作用 `extern` 占位函数 | `extern void PWM_SetDuty(uint8_t ch, uint16_t duty);` |
| **实战示例** | "怎么把电机转起来"（PWM配置/GPIO/ADC/接线） | 可用 STM32 HAL 等具体库 | `__HAL_TIM_SET_COMPARE(...)` |

**判断标准**：标题含"实战"或"配置"的，可用具体平台库；纯讲算法逻辑的，必须用平台无关的纯 C。

### 6.2 代码块 HTML 写法
```html
<div class="code-block"><span class="code-comment">// 注释</span>
<span class="code-keyword">void</span> <span class="code-func">Func</span>(<span class="code-keyword">void</span>) {
  <span class="code-keyword">return</span> <span class="code-number">0</span>;
}</div>
```
着色类：`code-comment`（灰）、`code-keyword`（紫）、`code-string`（绿）、`code-func`（蓝）、`code-number`（默认色）。`.code-block` 已设 `white-space: pre` 保留换行。

### 6.3 数学公式
块级 `$$...$$`，行内 `$...$`。模板字符串里反斜杠要双写（`\\sin` 渲染成 `\sin`）。

### 6.4 安全规则
- 推送前扫描邮箱/手机号/QQ/密钥（`1547357377`、`@qq.com`、`\b1[3-9]\d{9}\b` 等）
- `.claude/`、`*.log` 已在 `.gitignore`
- git 推送失败直接交给用户手动推，不纠结网络

## 七、如何添加 / 完善内容

### 任务 A：新增一个知识点（进阶篇/机器人篇）
1. `nav` 对应分组 `children` 加 `{ id: 'xxx', label: '标题' }`
2. 对应 `MotorData.{beginner|advanced|robotics}.sections` 加完整对象
3.（可选）`QuizData['xxx']` 加题，`nav` 自动显示"测验"标记
4. content 里如需图表，放 `<div data-chart="类型">` 占位
5. `node --check js/data.js` 语法检查 → 本地起服务点一遍 → 提交

### 任务 B：新增一种电机
1. `nav.motors.children` 加 `{ id: 'new-motor', label: '名称' }`
2. `MotorData.motorTypes` 加完整对象（title/subtitle/icon/overview/specs/sections）
3.（可选）`compareTable.rows` 加对比行；`QuizData['motor-new-motor']` 加题

### 任务 C：新增 ECharts 图表类型
1. `charts.js` 的 `_charts` 对象加一个函数（如 `mysignal(el) { echarts.init(el).setOption({...}) }`）
2. content 里放 `<div data-chart="mysignal" class="chart-container" style="height:300px"></div>`
3. 无需改 app.js（`renderAll` 自动调度）

### 任务 D：补充交互式 SVG 组件
1. `charts.js` 的 `_charts` 加函数，`el.innerHTML = '<svg>...</svg>'` + 绑定事件
2. content 里放 `<div data-chart="组件名" class="chart-container">` 占位
3. 颜色用 `var(--primary)` 等保证深色模式适配
