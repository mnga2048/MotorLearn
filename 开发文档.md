# MotorLearn 电机知识学习平台 · 开发文档

> 面向自动化学生 / 嵌入式开发者的电机控制知识学习网站。本仓库为**纯前端静态站点**，无需后端、无需构建，双击 `index.html` 即可运行。
>
> 本文档面向后续维护者（含 AI 协作者），说明：项目定位、技术选型思路、整体架构、内容数据层如何扩展，以及后续可补充的方向。

---

## 一、项目定位

| 项目        | 说明 |
|-----------|------|
| 名称        | MotorLearn（电机知识学习平台） |
| 目标用户    | 自动化 / 机电 / 电子类专业学生、嵌入式单片机（STM32/Arduino）开发者 |
| 核心价值    | 把分散在课本、博客、数据手册里的电机控制知识，**系统化、可视化、可检索**地组织在一个网页里 |
| 运行方式    | 纯静态：直接用浏览器打开 `index.html`，或任意 HTTP 服务器托管 |
| 是否需要后端| 否。所有数据持久化用浏览器 `localStorage` |

网站采用**单页应用（SPA）**模式：一个 `index.html` 外壳 + JS 动态渲染内容区。导航点击不刷新页面，靠 JS 切换内容。

## 二、技术栈与选型思路

### 选型原则
1. **零依赖、零构建**：不引入 React/Vue/Webpack，保证任何人 clone 下来就能改、就能跑。
2. **CDN 即用**：只引用必须的外部库（Tailwind / ECharts / KaTeX），全部走 CDN，不下载到本地。
3. **数据驱动**：所有内容写在 `js/data.js` 一个大对象里，UI 只负责把数据渲染成 HTML，互不耦合。

### 具体技术清单

| 类别 | 技术 | 用途 | 选型理由 |
|------|------|------|---------|
| 样式 | Tailwind CSS（CDN） | 原子化布局类 | 写得快、不用维护一堆 class 名 |
| 主题 | 原生 CSS 变量 | 暖色纸张主题 + 深色模式 | 切换主题只改 `data-theme`，变量自动替换 |
| 公式 | KaTeX + auto-render | 渲染 `$F=BIL$` 这类物理公式 | 电机控制绕不开公式，比 MathJax 快 |
| 图表 | ECharts 5 | 知识图谱（关系图）、进度饼图 | 中文文档好、关系图支持拖拽 |
| 交互 | 原生 JS | SPA 路由、手风琴、Tab、表单 | 不引框架，体量小、可控 |
| 存储 | localStorage | 收藏、进度、答题、主题偏好 | 无需后端的"用户态" |

### 脚本加载顺序（关键！）
`index.html` 底部按依赖顺序加载，**顺序不能乱**：

```
data.js       ← 内容数据（定义全局 MotorData、QuizData、AllKnowledgeIds）
  ↓
progress.js   ← 进度模块（依赖 MotorData）
favorites.js  ← 收藏 + 浏览历史（依赖 MotorData）
search.js     ← 搜索模块（依赖 MotorData）
calculator.js ← 公式计算器
charts.js     ← ECharts 图表（依赖 Progress）
validator.js  ← 协议校验工具
quiz.js       ← 自测练习模块（依赖 QuizData、navigateTo）
  ↓
app.js        ← 主控制器，最后加载，使用以上所有模块
```

> ⚠️ 历史教训：曾因 `quiz.js` 漏挂 `<script>` 标签，导致所有带测验的知识详情页抛 `ReferenceError` 白屏。新增 JS 模块时务必同步更新 `index.html` 的引用，并保持在 `app.js` **之前**。

## 三、整体架构

### 目录结构
```
电机学习/
├── index.html              # SPA 外壳：导航栏、侧边栏、内容容器、script 引用
├── css/
│   └── style.css           # CSS 变量、组件样式（手风琴、卡片、测验、校验工具）
├── js/
│   ├── data.js             # ⭐ 所有内容数据（1500+ 行，项目的"数据库"）
│   ├── app.js              # ⭐ 主控制器（路由、页面渲染、交互绑定）
│   ├── progress.js         # 学习进度（pending/learning/completed）
│   ├── favorites.js        # 收藏 + 浏览历史
│   ├── search.js           # 全文搜索 + 结果跳转 + 关键词高亮
│   ├── quiz.js             # 自测练习渲染 + 答题记录
│   ├── calculator.js       # 公式计算器（KV、步距角等）
│   ├── charts.js           # ECharts 知识图谱 / 进度饼图
│   └── validator.js        # CRC8/16/32、异或、取反校验工具
├── assets/                 # 图片资源（目前几乎为空，待补充）
└── 开发文档.md             # 本文档
```

### 运行时数据流

```
┌─────────────┐    渲染     ┌──────────────┐
│  data.js    │ ─────────▶ │   app.js     │ ──▶ innerHTML 注入 #page-container
│ (MotorData) │            │ (renderPage) │
└─────────────┘            └──────┬───────┘
      ▲                            │ 读写
      │                            ▼
      │                   ┌──────────────────┐
      └──── 读引用 ───────│ localStorage     │
                          │ (进度/收藏/答题)  │
                          └──────────────────┘
```

**核心渲染函数** `renderPage(pageId)`（位于 `app.js`）根据 `pageId` 分发：

| pageId | 渲染函数 | 示例 |
|--------|---------|------|
| `home` | `renderHomePage()` | 首页统计 + 快速入口 + 速览表 |
| `beginner` / `advanced` | `renderSectionPage()` | 章节卡片列表 |
| `motors` | `renderMotorsPage()` | 5 种电机卡片列表 |
| `industry` | `renderIndustryPage()` | 行业科普手风琴 |
| `roadmap` | `renderRoadmapPage()` | 知识图谱 + 书单 + 选型 |
| `tools` | `renderToolsPage()` | 对比 / 计算器 / 校验 / 进度 |
| `beginner-em` 等 | `renderDetailPage()` | 知识点详情（含测验） |
| `bldc` 等电机 id | `renderMotorPage()` | 电机详情（手风琴 + 测验） |

## 四、数据层结构（最常改的部分）

所有内容都在 `js/data.js` 的 `MotorData` 对象里。以下是关键字段，**添加新内容 = 往对应字段里追加对象**。

### 4.1 导航 `nav`
侧边栏的来源。支持一级菜单和可折叠的二级菜单：

```js
nav: [
  { id: 'home', label: '首页', icon: 'home' },                 // 一级
  {
    id: 'beginner', label: '入门篇', icon: 'book-open',
    badge: '入门', badgeClass: 'badge-beginner',
    children: [                                                  // 二级（可折叠）
      { id: 'beginner-em', label: '电磁学基础' },
      // ...
    ]
  },
]
```

> **添加一个新导航项后，必须同时在下面的数据结构里准备对应内容**，否则点击会回退到首页。

### 4.2 知识章节 `beginner.sections` / `advanced.sections`
每个 section 是一个知识点详情页：

```js
{
  id: 'beginner-em',                  // 必须与 nav.children.id 一致
  title: '电磁学基础',
  desc: '理解电机运转的物理原理',     // 列表卡片显示的简介
  icon: '⚡',
  tags: ['基础理论', '必学'],
  content: `                          // 详情页正文，支持 HTML
    <h3>电机运转的物理本质</h3>
    <p>所有电机的核心原理...</p>
    <div class="formula-block">$$F = BIL \\sin\\theta$$</div>
  `
}
```

**content 写法约定**：
- 用 HTML 标签，配合 Tailwind 类（`text-gray-600` 等）和自定义类（`.info-box`、`.formula-block`、`.compare-table`）
- 数学公式用 `$$...$$`（块级）或 `$...$`（行内），KaTeX 会自动渲染
- 代码示例用 `<pre><code>` 包裹

### 4.3 电机类型 `motorTypes`
每种电机是一个 key（与 nav.children.id 对应）：

```js
motorTypes: {
  'bldc': {
    title: '无刷直流电机 (BLDC)',
    subtitle: '现代电机控制的核心',
    icon: '🌀',
    overview: '一段较长的概述（用于详情页顶部信息框 + 搜索摘要）',
    specs: { 电压: '...', 转速: '...', 效率: '...' },   // 键值对，渲染成参数表
    sections: [                                          // 手风琴子章节
      { title: '工作原理', content: `<p>...</p>` },
      { title: '驱动方式', content: `<p>...</p>` },
    ]
  },
}
```

### 4.4 自测题 `QuizData`
题库独立于内容，按 section id 索引：

```js
const QuizData = {
  'beginner-em': [                    // key = section.id 或 'motor-' + 电机id
    {
      question: '通电导体在磁场中受到的力称为？',
      options: ['安培力', '法拉第力', '洛伦兹电动力', '库仑力'],
      answer: 0,                      // 正确选项的索引（0 起）
      explanation: '安培力公式 F = BIL...'  // 答题后显示的解析
    },
  ],
};
```

> 给某 section 加测验只需在 `QuizData` 里追加数组，详情页会自动渲染"自测练习"区。

### 4.5 其他数据
- `home`：首页统计数字、快速入口卡片
- `industry`：行业科普手风琴
- `compareTable`：电机对比表（headers + rows 二维数组）
- `roadmap`：学习路径（phases / books / devboards）
- `icons`：SVG 图标映射，供 nav 使用

## 五、如何添加 / 完善内容（常见任务）

### 任务 A：新增一个"入门篇"知识点
1. 在 `nav` 的 `beginner.children` 里加 `{ id: 'beginner-xxx', label: '标题' }`
2. 在 `MotorData.beginner.sections` 里加完整对象（id、title、desc、icon、content）
3.（可选）在 `QuizData['beginner-xxx']` 里加几道题
4. 保存，刷新浏览器，侧边栏即可看到新条目

### 任务 B：新增一种电机
1. 在 `nav` 的 `motors.children` 里加 `{ id: 'new-motor', label: '名称' }`
2. 在 `MotorData.motorTypes` 里加 `'new-motor': { title, subtitle, icon, overview, specs, sections }`
3.（可选）在 `compareTable.rows` 里加一行对比数据
4.（可选）在 `QuizData['motor-new-motor']` 里加题

### 任务 C：补充配图
1. 把图片放到 `assets/images/`
2. 在 content 里用 `<img src="assets/images/xxx.png" alt="...">` 引用
3. 若是示意图/原理图，建议配合文字说明，提升可读性

## 七、维护注意事项

1. **改数据优先于改逻辑**：90% 的内容更新只需动 `data.js`，不要去碰 `app.js` 的渲染逻辑。
2. **新增 JS 模块要同步挂 `<script>`**：在 `index.html` 的 `app.js` 之前加载，避免 ReferenceError。
3. **localStorage key 命名**：统一用 `motorlearn_` 前缀（`motorlearn_progress` / `motorlearn_favorites` 等）。
4. **样式优先用 CSS 变量**：颜色用 `var(--primary)` 等，不要硬编码，保证深色模式自动适配。
5. **提交前自测**：`node --check js/你的文件.js` 做语法检查；本地 `python -m http.server` 起服务，点一遍导航确认无白屏。
6. **数学公式转义**：content 是模板字符串，反斜杠要写双份（`\\sin` 才能渲染成 `\sin`）。
